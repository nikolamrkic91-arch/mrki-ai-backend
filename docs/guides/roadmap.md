### Submitting Feature Requests

To request a new feature:

1. Search existing [feature requests](https://github.com/mrki/mrki/issues?q=is%3Aissue+label%3Aenhancement)
2. If not found, create a new issue using the [feature request template](https://github.com/mrki/mrki/issues/new?template=feature_request.yml)
3. Describe the use case and expected behavior
4. Add the `enhancement` label

## Contributing to the Roadmap

We welcome community input on our roadmap:

- Vote on existing feature requests with 👍 reactions
- Comment on issues with additional context
- Submit pull requests for planned features
- Join discussions in our [community forum](https://github.com/mrki/mrki/discussions)

## Long-term Vision

Our goal is to make Mrki the most user-friendly and powerful workflow automation platform. Key focus areas:

1. **Ease of Use**: Intuitive interfaces for all skill levels
2. **Flexibility**: Support for diverse use cases and integrations
3. **Scalability**: Handle workflows of any size
4. **Reliability**: Enterprise-grade stability and security
5. **Community**: Foster an active contributor community
